# Extraction/evaluation-only boundary

This record is extraction/evaluation-only. It is not an application input.

# Decision record

| Observed class | Source-blind representation | Decision and boundary |
| --- | --- | --- |
| Repeated aligned headers and cells | Persistent header, column alignment, stable row rhythm | Retain the structural relationship, not labels or geometry. |
| Four recurring columns | Record state, primary record, identity context, grouped state summary | Generalize roles so they are understandable without product terminology. |
| Strongest record text | Primary-record hierarchy | Retain relative importance; omit identifiers and supporting field specifics. |
| Compact identity treatment | Secondary identity context | Retain the relationship; leave the representation and content product-owned. |
| Repeated compact indicators in one cell | Grouped state summary | Retain grouping and rhythm; do not assign semantic values, counts, or updates. |
| Individual-value links | Conditional in-cell link affordance | Preserve the narrow affordance only when a target product supplies a destination; reject row-click inference. |
| Neutral visual treatment and sparse accents | Restrained neutral contrast, light separation, limited accents | Preserve qualitative density and emphasis without palette, measurements, or source styling details. |
| Adjacent and wider page features | Explicitly unresolved and out of scope | Avoid turning excluded context into application requirements. |

# Application guardrails

The manifest does not supply display copy, product meaning, fixtures, state values, destinations, or interaction behavior. A later application must obtain those from current target-product implementation, applicable requirements, or an already named local fallback. Missing inputs remain unresolved.

# Reviewable inference boundary

The reading order and coherent-summary interpretation are inferences from recurring layout relationships. They are marked as such in the source-blind manifest. No inference is presented as a product requirement or as observed semantics.
