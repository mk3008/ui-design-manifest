# Implementer allowlist

Allowed inputs are:

- `design-manifest/`
- `application-input-contract.md`
- `apply-prompt.md`
- current target-product implementation and repository guidance
- explicit product requirements and supplied target-product data

# Prohibited inputs

Do not access or use extraction records, evaluation records, visual evidence, source-specific material, URLs, remembered source vocabulary, or unlisted files from this experiment.

# Delivery boundary

Apply only the source-independent relationships: a dedicated open vertical navigation region, ordered rows, optional product-supplied hierarchy, and optional product-supplied current-state treatment. Do not add display copy, controls, state, destinations, data, or interaction that the allowed inputs do not establish. Report omissions or unresolved inputs to the application owner.
