# Build

Using only this apply instruction, the Manifest, and the Application Input Contract, create one static comparison HTML plus captures for `A-open`, `A-hidden`, `B-open`, and `B-rail`. Keep the declared Header and navigation fixture identical; vary only control ownership and closed representation.

Pattern A retains the Header's leading named controller after the left Drawer is fully hidden. Pattern B has no Header Drawer trigger; its icon rail retains the named Drawer controller and its declared SVG shortcuts. Make sole-controller ownership obvious. Use square menu rows, a visible current-item accent, and no caption.

Use meaningful inline `currentColor` SVG icons. Never use bracket glyphs, text-character icon substitutes, empty boxes, or placeholder marks. Do not add behavior, breakpoints, animation, focus, keyboard handling, persistence, or any input not declared by the Contract.
