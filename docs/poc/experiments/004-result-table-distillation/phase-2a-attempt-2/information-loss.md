# Extraction/evaluation-only boundary

This record is extraction/evaluation-only. It is not an application input.

# Information-loss categories

| Category | Classification | Treatment |
| --- | --- | --- |
| Original identity and address | Intentionally excluded | Retained only in historical source-aware material; not needed to apply role relationships. |
| Visible wording and record data | Intentionally excluded | The manifest authorizes no display copy or fixture values. |
| Header wording and product vocabulary | Intentionally excluded | Generalized into role names that describe responsibility rather than domain meaning. |
| Screenshots, capture references, and evidence identifiers | Intentionally excluded | Extraction/evaluation-only material; never included in the application input. |
| Exact measurements, palette values, and implementation mechanics | Abstracted away | Replaced with relational direction for density, contrast, grouping, and emphasis. |
| State labels, semantics, values, data sources, and update triggers | Unresolved | Must come from the target product or requirements before state-bearing UI is added. |
| Link destinations and interaction details | Unresolved | Only the conditional, per-value affordance is retained. |
| Responsive, overflow, focus, and assistive-technology behavior | Unresolved | Outside the observed bounded slice and not invented by extraction. |
| Actions and broader page context | Explicitly out of scope | Not authorized by this pattern. |

# Sufficiency assessment

The retained direction is sufficient to guide a bounded structural treatment: hierarchy, density, grouping, reading order, relative importance, and responsibility boundaries. It is not sufficient to create product data, visible wording, state semantics, or interaction behavior. Those omissions are deliberate and must remain visible to a later implementer.
